# 外部内容合规政策

**文档用途**: 定义外部内容导入、存储和分享的合规边界  
**当前版本**: v0.1  
**最后更新**: 2026-04-29

---

## 一、核心原则

### 1.1 用户主动导入
- ✅ 用户手动输入链接或粘贴文本
- ✅ 用户主动触发保存操作
- ❌ 不做批量自动抓取
- ❌ 不做定时爬虫
- ❌ 不做无用户触发的后台抓取

### 1.2 尊重第三方平台权限
- ❌ 不绕过登录墙
- ❌ 不绕过付费墙
- ❌ 不绕过反爬机制
- ❌ 不绕过平台限制
- ✅ 只保存用户已有权限访问的内容

### 1.3 不公开展示第三方全文
- ❌ 不公开展示完整第三方文章
- ❌ 不对外提供第三方内容 API
- ✅ 用户私密保存和阅读
- ✅ 分享时只展示标题、来源、链接和用户笔记

## 二、功能边界

### 2.1 MVP 阶段允许的功能

**保存链接**:
- 用户输入 URL
- 用户手动编辑标题、来源、摘要
- 系统保存元数据（不抓取全文）
- 用户点击链接跳转到原网站阅读

**粘贴文本**:
- 用户复制粘贴文本内容
- 用户手动标注来源和作者
- 系统保存用户粘贴的内容
- 显示合规提示

**保存图片文字**:
- 用户上传图片
- 用户手动输入或编辑文字
- MVP 不做 OCR（未来可扩展）

**保存 PDF**:
- 用户上传 PDF
- 用户手动输入或编辑摘要
- MVP 不做 PDF 解析（未来可扩展）

### 2.2 MVP 阶段不允许的功能

**批量导入**:
- ❌ 不做"导入我的微信收藏"
- ❌ 不做"导入我的浏览器书签"
- ❌ 不做"同步我的 Pocket"
- ❌ 不做批量 URL 导入

**自动抓取**:
- ❌ 不做自动提取网页正文
- ❌ 不做自动提取标题和摘要
- ❌ 不做自动下载图片
- ❌ 不做自动解析 PDF

**绕过限制**:
- ❌ 不提供"去除广告"功能
- ❌ 不提供"绕过登录"功能
- ❌ 不提供"破解付费"功能
- ❌ 不提供"反反爬"功能

## 四、数据存储规范

### 4.1 external_items 表字段说明

**url** (text):
- 存储原始链接
- 用户点击时跳转到原网站
- 不存储抓取的 HTML

**title** (text):
- 用户手动输入或编辑
- MVP 不自动提取

**source_platform** (text):
- 用户手动输入（如"微信公众号"、"知乎"）
- MVP 不自动识别

**source_author** (text):
- 用户手动输入
- MVP 不自动提取

**excerpt** (text):
- 用户手动输入或编辑
- MVP 不自动提取

**original_content** (text):
- 仅存储用户粘贴的文本
- 不存储自动抓取的内容

**extracted_content** (text):
- MVP 阶段为 NULL
- 未来可用于 OCR 或 PDF 解析结果

**legal_note** (text):
- 自动生成合规提示
- 不可编辑

### 4.2 RLS 权限

```sql
-- 用户只能查看自己保存的外部内容
CREATE POLICY "Users can view own external items" ON external_items
  FOR SELECT USING (user_id = auth.uid());

-- 用户只能创建自己的外部内容
CREATE POLICY "Users can create own external items" ON external_items
  FOR INSERT WITH CHECK (user_id = auth.uid());

-- 用户只能更新自己的外部内容
CREATE POLICY "Users can update own external items" ON external_items
  FOR UPDATE USING (user_id = auth.uid());

-- 用户只能删除自己的外部内容
CREATE POLICY "Users can delete own external items" ON external_items
  FOR DELETE USING (user_id = auth.uid());
```

## 五、分享边界

### 5.1 允许的分享方式

**分享专题**:
- ✅ 展示专题名称和描述

- ✅ 展示外部内容元数据（标题、来源、原文链接）
- ✅ 展示用户的笔记和读后感
- ❌ 不展示外部内容全文

**分享笔记**:
- ✅ 展示笔记内容
- ✅ 展示关联文章的标题和链接
- ✅ 展示外部内容的标题、来源和原文链接
- ❌ 不展示外部内容全文

**分享读后感**:
- ✅ 展示读后感内容
- ✅ 展示关联文章的标题和链接
- ✅ 展示外部内容的标题、来源和原文链接
- ❌ 不展示外部内容全文

### 5.2 不允许的分享方式

- ❌ 不提供外部内容全文的公开链接
- ❌ 不提供外部内容的 RSS 订阅
- ❌ 不提供外部内容的 API 导出
- ❌ 不允许搜索引擎索引外部内容全文

## 六、未来扩展边界

- ✅ OCR 图片文字识别（用户上传的图片）
- ✅ PDF 文本提取（用户上传的 PDF）

- 必须是用户主动触发

- ❌ 公开内容聚合

- ❌ 内容分发网络
- ❌ 反爬虫工具
- ❌ 付费墙破解工具

## 七、风险控制

### 7.1 技术控制

**频率限制**:
- 每个用户每天最多保存 100 个外部内容
- 每个用户每小时最多保存 20 个外部内容

**内容审核**:
- 用户举报机制
- 管理员人工审核
- 发现违规内容及时下架

**访问控制**:
- 外部内容默认私密
- 不提供公开访问链接
- 不允许搜索引擎爬取

### 7.2 法律风险应对

**用户协议**:
- 明确告知用户合规要求
- 用户承诺遵守第三方平台服务条款
- 用户承诺不侵犯版权

**免责声明**:
- 平台不对用户保存的外部内容负责
- 平台不对第三方内容的合法性负责
- 平台仅提供个人资料整理工具

**投诉机制**:
- 提供版权投诉渠道
- 及时响应投诉
- 及时删除侵权内容

## 八、开发检查清单

### 8.1 功能开发前检查

在开发任何外部内容相关功能前，必须回答：

1. **是否用户主动触发？**
   - 如果不是，停止开发

2. **是否绕过第三方限制？**
   - 如果是，停止开发

3. **是否公开展示第三方全文？**
   - 如果是，停止开发

4. **是否批量自动抓取？**
   - 如果是，停止开发

5. **是否符合合规原则？**
   - 如果不符合，停止开发

### 8.2 功能上线前检查

1. ✅ 是否显示合规提示？
2. ✅ 是否限制访问权限（仅用户可见）？
3. ✅ 是否限制分享范围（不展示全文）？
4. ✅ 是否添加频率限制？
5. ✅ 是否添加举报机制？

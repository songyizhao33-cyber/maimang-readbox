# MVP Demo Script

Target length: 5-8 minutes.

Use test accounts only. Do not use real passwords, real private reading data, or production user records in a recording.

## Demo Path

1. Open the app homepage.
   - Say: "Maimang Readbox is a quiet reading space for subscriptions, saved external reading, private notes, and long-term organization."

2. Sign in with a test reader account.
   - Avoid showing real credentials.

3. Open `Authors`.
   - Show public author cards.
   - Explain that the list is not ranked and is not a recommendation feed.

4. Subscribe to an author.
   - Show the Subscribe / Subscribed state.
   - Explain that future published articles from this author enter Inbox.

5. Open `Inbox`.
   - Show subscription-delivered articles.
   - Briefly show Unread, Reading, Starred, and Archived semantics.

6. Open a published article.
   - Show title, author, published date, body, and source navigation.
   - Point out public reading traces as read-only notes/reflections, not comments.

7. Add a private note.
   - Use a short selected quote.
   - Explain that notes are fragmentary reading marks.

8. Add a private reflection.
   - Explain that reflections are longer thoughts after reading.
   - Mention that public visibility is only for article traces and stays controlled.

9. Open `Later`.
   - Save an external item manually with title, source URL, source platform, author name, and excerpt.
   - Explain that the app does not automatically import pages or expose third-party full text.

10. Open the external item detail page.
    - Show source metadata and compliance note.
    - Add an external item note and reflection.
    - Explain that external item traces remain in the private workspace in this MVP.

11. Open `Collections`.
    - Create a private collection.
    - Add the saved external item to it from Later.
    - If a prepared article relation exists, show article and external item together in collection detail.

12. Open collection detail.
    - Explain "Remove from collection" only removes the relation and does not delete the original item.

13. Open `Reading Traces`.
    - Show aggregated notes and reflections.
    - Use filters: All, Notes, Reflections, Articles, External items.
    - Click View source to return to the article or external item.

14. Open `Settings`.
    - Show safe profile fields: display name, bio, avatar URL, role.
    - Mention that email and internal IDs are not shown.

15. Open `Author Dashboard`.
    - If no author profile exists, show the create-author explanation.
    - If an author profile exists, show Write, My Articles, and Public Author Page links.

16. Open `Write` and `My Articles`.
    - Show draft creation flow.
    - Explain that drafts remain private until published.
    - Show Draft / Published badges.

17. Close with explicit MVP boundaries.
    - No recommendation feed.
    - No trending list.
    - No comments or likes.
    - No search or tags yet.
    - No AI summary, OCR, or automatic crawling.
    - External items are manually saved private records.

## Suggested Closing Line

"The MVP is not trying to maximize attention. It gives readers a quiet inbox, a manual later shelf, private collections, and a durable trace of what they read and thought."

## 九、自我反省问题

### 9.1 产品定位检查

**是否偏离"小而美"定位？**
- [ ] 是否添加了推荐算法？
- [ ] 是否添加了热榜排行？
- [ ] 是否添加了评论系统？
- [ ] 是否添加了私信功能？
- [ ] 是否添加了复杂社交功能？

**如果是，立即停止并删除相关代码。**

### 9.2 过度工程检查

**是否引入了不必要的复杂度？**
- [ ] 是否为单次使用创建了抽象？
- [ ] 是否引入了重型依赖库？
- [ ] 是否实现了未要求的功能？
- [ ] 是否为"未来可能需要"提前设计？
- [ ] 是否过度使用设计模式？

**如果是，简化代码。**

### 9.3 外部内容合规检查

**是否违反了外部内容合规原则？**
- [ ] 是否实现了批量爬虫？
- [ ] 是否绕过了登录墙/付费墙？
- [ ] 是否公开展示了第三方全文？
- [ ] 是否自动抓取了第三方内容？
- [ ] 是否提供了反爬虫工具？

**如果是，立即停止并删除相关代码。**

### 9.4 模块边界检查

**是否所有模块都有清晰边界？**
- [ ] 每个模块职责是否单一？
- [ ] 模块之间依赖是否清晰？
- [ ] 是否有循环依赖？
- [ ] 是否有模块职责重叠？

**如果有问题，重新划分模块。**

### 9.5 接口契约检查

**是否所有接口都有契约？**
- [ ] 每个接口是否有完整定义？
- [ ] 接口设计是否合理？
- [ ] 接口是否符合 RESTful 规范？
- [ ] 接口权限是否正确？

**如果有问题，更新接口契约。**

### 9.6 任务验收标准检查

**是否所有任务都有验收标准？**
- [ ] 每个任务是否有明确验收标准？
- [ ] 验收标准是否可验证？
- [ ] 验收标准是否完整？
- [ ] 是否有验证命令？

**如果没有，补充验收标准。**

## 十、阶段性反省时机

### 10.1 每日反省
- 今天完成了哪些任务？
- 是否偏离了项目目标？
- 是否引入了不必要的复杂度？
- 明天计划做什么？

### 10.2 每周反省
- 本周完成了哪些 Phase？
- 是否按计划推进？
- 遇到了哪些问题？
- 下周计划做什么？

### 10.3 Phase 完成后反省
- 这个 Phase 的目标是否达成？
- 是否所有任务都通过验收？
- 是否有遗留问题？
- 下一个 Phase 准备好了吗？

### 10.4 MVP 完成前反省
- 是否所有核心功能已实现？
- 是否通过了所有检查点？
- 是否符合产品定位？
- 是否可以交付？
